# DataChat — CSV Data Analysis Agent

> **All commands run from the repo root** (`ai-spec-driven-boilerplate/`).

Upload a CSV file and ask natural language questions about your data. Answers are generated by Google Gemini. Runs in stub mode (no API key required) for development and testing.

---

## Quick Start

### 1. Install dependencies

```bash
# from: ai-spec-driven-boilerplate/
uv sync
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and set:

```
DATACHAT_DATABASE_URL=sqlite:///./datachat.db
GEMINI_API_KEY=your-key-here          # leave blank for stub mode
DATACHAT_LLM_MODEL=gemini-2.5-flash
DATACHAT_UPLOAD_DIR=uploads
```

### 3. Apply database migrations

```bash
# from: ai-spec-driven-boilerplate/
uv run alembic upgrade head
uv run alembic current   # must print a revision hash — blank means it failed
```

### 4. Run the server

```bash
# from: ai-spec-driven-boilerplate/
uv run python -m datachat
```

Open [http://localhost:8001](http://localhost:8001) in your browser.

---

## Using the App

1. **Upload a CSV** — click "Upload your CSV", pick a `.csv` file, click Upload.
2. **Ask a question** — type a natural language question about your data (e.g. "What is the average age?") and click Ask.
3. **Read the answer** — the answer appears below the question box.

If `GEMINI_API_KEY` is not set, the app runs in **Stub Mode** — a yellow banner appears on every page and answers are placeholder text. Set the key and restart to use real Gemini.

---

## Running Tests

```bash
# from: ai-spec-driven-boilerplate/
uv run pytest
```

Tests run fully offline — no `GEMINI_API_KEY` required. All tests use a stub LLM and an in-memory SQLite database.

## Project Layout

```
src/datachat/
  api/          ← FastAPI routes (uploads, queries, pages, health)
  config/       ← Pydantic settings (env prefix: DATACHAT_)
  db/           ← SQLAlchemy models + session
  domain/       ← Pydantic domain models (Upload, Query)
  llm/          ← LLM provider layer (Gemini + stub)
  pipeline/     ← CSV reader + query runner
  templates/    ← Jinja2 HTML template
tests/
  unit/         ← Model and smoke tests
  integration/  ← Golden-path end-to-end tests
alembic/        ← DB migrations
```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Health check |
| `GET` | `/` | Web UI |
| `POST` | `/api/uploads` | Upload a CSV (multipart) |
| `GET` | `/api/uploads/{id}` | Get upload metadata |
| `POST` | `/api/queries` | Ask a question (`upload_id` + `question`) |
| `GET` | `/api/queries/{id}` | Get a saved query and answer |

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `DATACHAT_DATABASE_URL` | `sqlite:///./datachat.db` | SQLAlchemy DB URL |
| `GEMINI_API_KEY` | *(empty)* | Google Gemini API key — leave blank for stub mode |
| `DATACHAT_LLM_MODEL` | `gemini-2.5-flash` | Gemini model name |
| `DATACHAT_UPLOAD_DIR` | `uploads` | Directory where CSV files are stored |
| `DATACHAT_LOG_LEVEL` | `INFO` | Log level |
