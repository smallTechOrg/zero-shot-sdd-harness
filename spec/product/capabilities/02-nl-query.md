# Capability: Natural Language Query

**What it does:** Accept a natural-language question and an `upload_id`, load the corresponding CSV from disk, construct a context prompt with column names and sample rows, call the Google Gemini API, and return the plain-text answer — also storing the question and answer in SQLite.

---

## Inputs

| Input | Source | Type | Constraints |
|-------|--------|------|-------------|
| `upload_id` | JSON request body | UUID string | Must reference an existing record in the `uploads` table |
| `question` | JSON request body | String | Must be non-empty and non-whitespace; no maximum length enforced in v0.1 |

## Outputs

| Output | Type | Description |
|--------|------|-------------|
| `id` | UUID string | Unique identifier for this query record |
| `upload_id` | UUID string | Echo of the input `upload_id` |
| `question` | String | Echo of the input question |
| `answer` | String | Plain-text answer generated by Gemini |
| `created_at` | ISO 8601 datetime (UTC) | Timestamp of when the answer was stored |

## Processing Steps

1. Receive JSON request at `POST /api/queries`
2. Validate `upload_id` and `question` are present and non-empty
3. Look up the `Upload` record in SQLite; return 404 if not found
4. Construct the on-disk path `uploads/<upload.filename>` and verify the file exists
5. Read the CSV with `pandas.read_csv()`; take the first 20 rows as sample data
6. Build the Gemini prompt (see Prompt Structure below)
7. Call `genai.GenerativeModel.generate_content(prompt)`
8. Extract the answer text from the Gemini response
9. Generate a UUID4 for the query; insert a row into the `queries` SQLite table
10. Return the query record as JSON

## Prompt Structure

```
You are a data analyst assistant. The user has uploaded a CSV file with the following structure:

Columns: {comma-separated column names}

Sample data (first {N} rows):
{CSV text of first N rows, including header}

Answer the following question based on the data above. Be concise and specific.
If the answer cannot be determined from the sample data alone, say so clearly.

Question: {user's question}
```

The sample is capped at 20 rows in v0.1 to stay within typical token limits. This is a known limitation — large CSVs are only partially represented in the prompt.

## External Calls

| Call | Purpose |
|------|---------|
| SQLAlchemy `session.get(Upload, upload_id)` | Load upload metadata |
| `pandas.read_csv()` | Re-read CSV from disk to extract sample rows |
| `google.generativeai.GenerativeModel.generate_content()` | Send prompt to Gemini and receive answer |
| SQLAlchemy `session.add()` + `session.commit()` | Store query + answer in SQLite |

## Error Cases

| Error | Trigger | HTTP Status | Response |
|-------|---------|-------------|----------|
| Missing `upload_id` or `question` | Malformed request body | 422 | Pydantic validation error (FastAPI default) |
| Empty `question` | Whitespace-only string | 400 | `{"detail": "Question must not be empty"}` |
| Upload not found | `upload_id` not in DB | 404 | `{"detail": "Upload not found"}` |
| CSV file missing from disk | File deleted after upload | 500 | `{"detail": "CSV file not found on disk — please re-upload"}` |
| Gemini key not configured | `GEMINI_API_KEY` env var absent | 503 | `{"detail": "Gemini API key not configured. Set GEMINI_API_KEY and restart."}` |
| Gemini API error / quota exceeded | `google.api_core.exceptions` | 503 | `{"detail": "Gemini API error: <error message>"}` |
| Database error on query insert | SQLAlchemy exception | 500 | `{"detail": "Database error"}` |

## Known Limitations (v0.1)

- Only the first 20 rows of the CSV are sent to Gemini. Questions that require full-dataset aggregation (e.g., exact totals over thousands of rows) may produce incorrect answers. This is documented in the stub-mode banner and will be addressed in v0.2 with a pandas-computed context strategy.
- No conversation history — each question is answered independently with no memory of prior questions in the same session.
- No streaming — the answer arrives in one HTTP response after Gemini completes generation.

## Success Criteria

- [ ] Asking "how many columns does this file have?" against a 5-column CSV returns an answer containing the number 5
- [ ] Asking "what are the column names?" returns all column names mentioned in the answer
- [ ] A query record appears in the `queries` SQLite table with the correct `upload_id`, `question`, and `answer`
- [ ] Sending a request with a non-existent `upload_id` returns 404
- [ ] Sending a request when `GEMINI_API_KEY` is not set returns 503 with the key-not-configured message
- [ ] Sending an empty `question` field returns 400
