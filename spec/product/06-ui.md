# UI

## UI Type

Single-page web app served by FastAPI. Built with plain HTML and vanilla JavaScript — no framework, no build step, no npm. The entire frontend is a single `index.html` file (plus optional inline `<style>` and `<script>` blocks). FastAPI serves it at `GET /`.

## Views / Screens

### Screen: Main Page (single view)

**Purpose:** The entire user experience lives on one page. The user uploads a CSV, sees the detected columns, then asks questions and reads answers — all without navigating away.

**Layout:** Two vertically stacked panels inside a centered container.

---

#### Panel 1 — Upload Panel

**Purpose:** Let the user pick and upload a CSV file, then confirm success and display what was found.

**Key elements:**
- Stub-mode banner (shown at top of the full page when `GEMINI_API_KEY` is not set): yellow warning box reading "Gemini API key not configured — answers will not be generated. Set the GEMINI_API_KEY environment variable and restart the server."
- `<input type="file" accept=".csv">` file picker
- "Upload" button (disabled until a file is selected)
- Status area below the button:
  - Idle: empty
  - Loading: "Uploading..." spinner/text
  - Success: green confirmation showing original filename, row count, and a comma-separated list of column names
  - Error: red error message with the detail returned by the API

**Actions available:**
- Select a CSV file from disk
- Click "Upload" to POST the file to `/api/uploads`
- After success, the upload panel stores the `upload_id` in JS memory so the query panel can reference it

---

#### Panel 2 — Query Panel

**Purpose:** Let the user type a natural-language question about the uploaded CSV and display the answer.

**Key elements:**
- Disabled/greyed-out state before a CSV has been uploaded successfully (shows placeholder text: "Upload a CSV file first")
- `<textarea>` for the question (enabled once an upload succeeds)
- "Ask" button (disabled when textarea is empty or an upload has not been completed)
- Answer area below the button:
  - Idle: empty
  - Loading: "Thinking..." text
  - Success: the plain-text answer returned by Gemini, displayed in a readable block
  - Error: red error message with the detail from the API (e.g., "Gemini API quota exceeded")

**Actions available:**
- Type a natural-language question
- Click "Ask" to POST to `/api/queries` with the current `upload_id` and the question text
- The answer area clears and shows "Thinking..." while the request is in flight

---

## Error States

| Situation | UI Behavior |
|-----------|-------------|
| Gemini API key not set | Yellow stub-mode banner shown at top of page on load (detected via a lightweight `GET /api/status` endpoint or by attempting a query and receiving 503) |
| Upload fails (not a CSV, empty file) | Red error text replaces the upload status area; "Upload" button re-enabled |
| Network error during upload or query | Red error: "Network error — is the server running?" |
| Query fails (Gemini error or quota) | Red error text in the answer area; "Ask" button re-enabled |
| CSV file missing from disk at query time | Red error: "The uploaded file could not be found on the server. Please re-upload your CSV." |
| Loading (upload or query in progress) | Relevant button shows disabled + spinner text; prevents double-submit |

## Tech Stack

- Plain HTML5 + CSS (no framework)
- Vanilla JavaScript (ES2020, no transpilation)
- `fetch` API for HTTP calls
- No npm, no webpack, no build step
- Served by FastAPI via `StaticFiles` mount or an inline route returning the HTML string
