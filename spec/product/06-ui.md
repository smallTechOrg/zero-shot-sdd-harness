# UI

## UI Type

Browser-based single-page application (SPA) with a chat interface. The layout is a two-panel design: a left sidebar for session/file info and a main panel containing the chat thread.

## Tech Stack

React 18 + Vite 5 + Tailwind CSS 3. No component library — plain Tailwind utility classes. Axios for HTTP requests. No React Router needed (single-page, no routing in v0.1).

## Views / Screens

### Screen: Upload / Landing

**Purpose:** The entry point. The user has not yet uploaded a file. They are prompted to upload a CSV or JSON file to start a session.

**Key elements:**
- Centered card with a drag-and-drop file drop zone (click to browse or drag a file onto it)
- Accepted file types label: "CSV or JSON files up to 50 MB"
- A "Upload and Start" button (disabled until a file is selected)
- Upload progress indicator (spinner or progress bar) shown while the file is being sent and parsed
- Error banner below the drop zone for validation errors (wrong format, too large, parse failure)

**Actions available:**
- Select a file via file browser
- Drag and drop a file
- Click "Upload and Start" to submit — transitions to the Chat screen on success

---

### Screen: Chat

**Purpose:** The main interaction screen. The file has been uploaded successfully. The user can ask questions and see answers.

**Key elements:**
- **Left sidebar (fixed width ~260px):**
  - File info card: filename, row count, column count
  - Column list: scrollable list of column names and their dtypes
  - "Upload a new file" button — clears the session and returns to the Upload screen
- **Main chat panel:**
  - Chat thread — scrollable list of message bubbles, oldest at top, newest at bottom
    - User messages: right-aligned, blue background
    - Assistant messages: left-aligned, white/grey background
  - Each assistant message has an expandable "Reasoning trace" section (collapsed by default) showing the ordered list of THINK / ACTION / OBSERVE steps
  - Message input bar pinned to the bottom: a text area (auto-grows, max 4 rows) + "Send" button
  - Send button disabled while the agent is processing
  - Loading indicator in the chat thread (animated dots) while awaiting agent response

**Actions available:**
- Type a question and press Enter or click "Send"
- Expand/collapse the reasoning trace on any assistant message
- Scroll through chat history
- Click "Upload a new file" to start a new session

---

### Screen: Re-upload Required (inline state)

**Purpose:** Shown in the chat panel when the session's DataFrame is no longer in memory (server restarted). Not a separate screen — it is an inline banner above the message input bar.

**Key elements:**
- Yellow warning banner: "Your session data was cleared (server restarted). Please re-upload your file to continue chatting."
- "Re-upload file" button inside the banner — opens a file picker pre-populated with the original filename if possible

**Actions available:**
- Click "Re-upload file" — opens file picker, re-uploads, and restores the chat session

## Error States

| Scenario | UI Treatment |
|----------|-------------|
| File too large (> 50 MB) | Red error banner on Upload screen: "File exceeds the 50 MB limit. Please upload a smaller file." |
| Wrong file type | Red error banner: "Only CSV and JSON files are supported." |
| File parse failure | Red error banner with the parse error detail returned by the backend |
| Question with no active session | Disabled message input; prompt user to upload a file |
| Gemini API failure (503) | Red inline error in the chat thread: "The AI service is temporarily unavailable. Please try again." Message input re-enabled immediately. |
| Server error (500) | Red inline error in chat thread: "An unexpected error occurred. Please try again." |
| Session not found (404) | Full-screen error card with "Session not found" and a "Start over" button that returns to Upload screen |
| DataFrame gone (410) | Yellow inline banner — see "Re-upload Required" screen above |
| Question too long | Character counter below the text input turns red at > 2000 chars; Send button disabled |
| Network offline | Toast notification: "No network connection. Please check your connection and retry." |

## Loading States

- **File upload in progress:** Spinner + "Uploading and parsing your file…" text replaces the drop zone
- **Agent thinking:** Animated three-dot ellipsis in the chat thread (appears as an assistant message placeholder) while the ReAct loop runs
- **Message send:** Send button shows a spinner and is disabled; text input is disabled
