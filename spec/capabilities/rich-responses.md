# Capability: Rich Responses

## What It Does

Composes a structured analyst response from a DuckDB query result — a markdown narrative, a sortable data table, and an auto-selected Chart.js chart — and delivers it progressively over SSE to the browser for real-time rendering.

## Inputs

| Input | Type | Source | Required |
|-------|------|--------|----------|
| `query_result` | `QueryResultModel` | `execute_query` node (from DuckDB) | Yes (for data queries) |
| `question` | string | `AnalystState.question` | Yes |
| `intent` | string | `classify_intent` node | Yes |
| `sql` | string | `call_llm_with_tools` node | No (absent for off-topic) |
| `query_error` | string | `execute_query` node | No (present only on DuckDB error) |

## Outputs

| Output | Type | Destination |
|--------|------|-------------|
| SSE `chunk` events | Incremental markdown text tokens | SSE stream → frontend `ChatThread` |
| SSE `table` event | `QueryResultModel` JSON (columns, rows, row_count) | SSE stream → frontend `DataTable` |
| SSE `chart` event | `ChartSpec` JSON (type, labels, datasets) | SSE stream → frontend `AnalystChart` |
| `RichResponseModel` | Pydantic model serialized to JSON | Stored in `messages.content` (SQLite) |

## External Calls

| System | Operation | On Failure |
|--------|-----------|------------|
| Gemini 2.5 Flash | JSON-mode call to generate narrative text + ChartSpec from query result | Fallback: use query result shape to generate minimal narrative without chart; do not fail the whole response |

## Business Rules

- Narrative is generated by Gemini from the query result column names and rows — not from raw dataset rows.
- The prompt to Gemini for narrative generation includes the question, the SQL (for context), and the full query result (columns + rows up to 500). This is the query result, not the raw dataset — row count is bounded.
- Chart auto-selection is deterministic (no LLM call for chart type decision):
  - If result has exactly 1 numeric column and ≤20 rows and no date column → **bar** chart
  - If result has a DATE or TIMESTAMP column → **line** chart (date on X-axis)
  - If result has exactly 2 columns (any + numeric) and ≤8 rows → **pie** chart
  - Otherwise → no chart; table only
- If `intent == "off_topic"` or `intent == "clarification"`, the narrative is a canned string; no table or chart is emitted.
- If `query_error` is set, the narrative is a user-friendly error message ("The query could not be executed: {error}. Try rephrasing your question."); no table or chart is emitted.
- The `sql` field is included in `RichResponseModel` for display in the UI SQL attribution tooltip.
- SSE event order is always: `status` events → `chunk` events (streaming text) → `table` event → `chart` event → `done` event.
- `chunk` events contain partial markdown text tokens as they are generated (streaming). The frontend concatenates them to form the full narrative.

## Success Criteria

- [ ] An aggregation query (e.g. GROUP BY) returns a narrative, a data table, and a bar chart in the SSE stream.
- [ ] A time-series query (date column present) produces a line chart.
- [ ] A query returning > 500 rows shows "Showing 500 of {N} rows" in the DataTable footer.
- [ ] An off-topic question produces only a narrative response (no table or chart SSE events).
- [ ] A DuckDB error produces a user-friendly error narrative, no table, no chart, and `status=failed`.
- [ ] The narrative text streams progressively (multiple `chunk` events) — the user sees text appearing in real time.
- [ ] If Gemini's JSON mode call fails, the response still completes with a plain-text narrative (chart absent) — no 500 error.
- [ ] The `sql` field in `RichResponseModel` matches exactly the SQL that was executed.
