# API

## API Style

REST + Server-Sent Events (SSE). All JSON responses use the envelope `{"data": <payload>, "error": null}` on success or `{"data": null, "error": {"code": "...", "message": "..."}}` on failure, consistent with the existing `src/api/_common.py` helpers.

The SSE chat endpoint uses `text/event-stream`. All other endpoints return `application/json`.

Base URL: `http://localhost:8001` (dev). Frontend calls these from the same origin (no CORS needed in production mode; add CORS middleware only for `pnpm dev` inner-loop development).

## Authentication

None. Single-user local tool. Session identity is carried by the client-provided `session_id` (a UUID generated by the server at session creation and stored in browser `localStorage`).

### `POST /datasets`

**Purpose:** Upload a dataset file (CSV, Excel .xlsx, or JSON) to a session.

**Request:** `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `session_id` | string (UUID) | Yes | Target session |
| `file` | file | Yes | CSV, XLSX, or JSON file; max 100 MB |

**Response `201`:**
```json
{
  "data": {
    "dataset_id": "abc123",
    "name": "sales.csv",
    "row_count": 1234,
    "columns": [
      {"name": "date", "type": "DATE"},
      {"name": "revenue", "type": "DOUBLE"}
    ],
    "uploaded_at": "2026-06-23T10:05:00Z"
  },
  "error": null
}
```

**Behaviour:**
1. Validate file extension (`.csv`, `.xlsx`, `.json`).
2. Save file to `data/uploads/<session_id>/<filename>`.
3. Open DuckDB in-memory, load the file, infer schema (column names + DuckDB types), count rows.
4. Persist `Dataset` record to SQLite with `columns_json` and `row_count`.
5. Return the schema to the frontend.

**Error cases:**
| Status | Condition |
|--------|-----------|
| 400 | Unsupported file type; file too large (>100 MB); DuckDB failed to parse file |
| 404 | Session not found |
| 422 | Missing `session_id` or `file` field |
| 500 | Filesystem write failure |

### `GET /chat`

**Purpose:** Ask a natural-language question about the session's datasets. Returns a Server-Sent Events stream.

**Query params:**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `session_id` | string (UUID) | Yes | Target session |
| `q` | string | Yes | The natural-language question |

**Response headers:**
```
Content-Type: text/event-stream
Cache-Control: no-cache
X-Accel-Buffering: no
```

**SSE event sequence:**

```
event: status
data: {"node": "classify_intent", "message": "Analysing question..."}

event: status
data: {"node": "build_schema_context", "message": "Loading schema..."}

event: status
data: {"node": "call_llm_with_tools", "message": "Generating SQL..."}

event: status
data: {"node": "execute_query", "message": "Running query..."}

event: chunk
data: {"text": "The top regions by revenue are..."}

event: chunk
data: {"text": " led by North America with $2.3M."}

event: table
data: {"columns": ["region", "revenue"], "rows": [["North America", 2300000], ["EMEA", 1800000]], "row_count": 5}

event: chart
data: {"type": "bar", "labels": ["North America", "EMEA", "APAC"], "datasets": [{"label": "Revenue", "data": [2300000, 1800000, 900000]}]}

event: done
data: {"message_id": "msg-uuid-2", "status": "completed"}
```

**On error:**
```
event: error
data: {"message": "DuckDB error: column 'revnue' not found. Please check your question.", "node": "execute_query"}

event: done
data: {"message_id": "msg-uuid-2", "status": "failed"}
```

**Notes:**
- The `chunk` events carry incremental markdown text tokens. The frontend concatenates them.
- The `table` event carries the full query result (capped at 500 rows).
- The `chart` event carries the Chart.js-compatible spec. It is absent if no chart is appropriate.
- The `done` event always fires last, whether success or error.
- The message (user question + assistant response) is persisted to SQLite after `done`.

**Error cases (non-SSE — before stream opens):**
| Status | Condition |
|--------|-----------|
| 400 | Missing `session_id` or `q` |
| 404 | Session not found |
| 422 | `q` is blank |

### `GET /audit`

**Purpose:** List query audit log entries for a session.

**Query params:**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `session_id` | string (UUID) | Yes | Target session |
| `limit` | int | No | Max entries to return (default: 50, max: 200) |
| `offset` | int | No | Pagination offset (default: 0) |

**Response `200`:**
```json
{
  "data": {
    "total": 12,
    "entries": [
      {
        "query_log_id": "ql-uuid",
        "dataset_name": "sales.csv",
        "sql": "SELECT region, SUM(revenue) FROM sales GROUP BY region ORDER BY 2 DESC LIMIT 5",
        "row_count": 5,
        "latency_ms": 42,
        "error": null,
        "created_at": "2026-06-23T10:06:04Z"
      }
    ]
  },
  "error": null
}
```

**Error cases:**
| Status | Condition |
|--------|-----------|
| 404 | Session not found |
